# Архитектура ИИ: 5 Классов LSTM

В данном документе описана архитектура нейросети (на базе ActionLSTM) и логика определения действий, в частности, нового функционала — детекции кражи из витрины.

## Обзор
Система использует связку YOLOv8-Pose (для извлечения 17 ключевых точек скелета) и кастомной модели ActionLSTM (`libs/ai_models.py`). 
LSTM анализирует последовательность из 30 кадров и предсказывает, какое действие совершает человек.

## 5 Классов ИИ

| ID | Название | Описание | Задержка (Cooldown) | Порог уверенности |
|---|---|---|---|---|
| **0** | `Normal` | Нормальное поведение, игнорируется системой. | Нет | Нет |
| **1** | `Fight` | Драка (быстрые взмахи, удары, высокая динамика). | 20 сек | 0.60 |
| **2** | `Bullying` | Буллинг (медленные толчки, зажатие в угол, агрессивная стойка). | 30 сек | 0.60 |
| **3** | `Theft` | Кража из кассы. Длительная вытянутая рука к кассе. Проверяется попадание в **красную зону** (`theft_zone`). | 40 сек | 0.50 |
| **4** | `Shoplifting` | Кража из витрины. Рука тянется к полке и быстро прячется к телу/в карман. Проверяется попадание в **голубую зону** (`display_zone`). | 30 сек | 0.55 |

## Архитектура функционала "Кража из витрины" (Class 4)

Новый класс (ID 4) был добавлен для решения проблемы воровства товаров с полок. В отличие от кассы (где рука просто долго вытянута), кража с полки имеет специфический паттерн: рука тянется к витрине и **резко возвращается** обратно к телу (чтобы спрятать товар).

### 1. База Данных (`libs/models.py`)
Таблица `cameras` содержит два поля для этого функционала:
* `detect_shoplifting` (Boolean): Флаг включения/выключения детектора для конкретной камеры.
* `display_zone` (Text): JSON-строка, хранящая нормализованные координаты `[[x, y], [x, y], ...]` полигона Защитной Зоны витрины.

### 2. Конвейер (Worker Pipeline) (`apps/worker/pipeline.py`)
Когда LSTM предсказывает действие класса 4 (`action_class == 4`), пайплайн выполняет следующие шаги:
1. **Проверка флага**: Убеждается, что `cam.detect_shoplifting == True`.
2. **Проверка Защитной Зоны (ROI)**: Использует алгоритм `cv2.pointPolygonTest`. Система берет центр обнаруженного человека (`cx, cy`) и проверяет, находится ли он внутри заданного голубого полигона витрины (`display_zone`). Если человек находится за пределами полигона — детекция отбрасывается для защиты от ложных срабатываний.
3. **Генерация события**: Если все проверки пройдены, создается событие типа `shoplifting_suspected`, которое записывается в базу, и запускаются вебхуки.

### 3. Веб-интерфейс (UI)
Для настройки зоны витрины используется HTML5 Canvas на странице управления камерой (`camera_detail.html`):
* Полигон витрины отрисовывается **голубым цветом** (`#06B6D4` / cyan), чтобы визуально отличаться от зоны кассы (которая рисуется красным).
* Кнопка для рисования зоны витрины появляется только если активирован соответствующий чекбокс.

### 4. Обучение Модели (`scripts/train_lstm.py`)
При отсутствии реальных видеозаписей, скрипт `train_lstm.py` может генерировать синтетические данные для обучения. 
Паттерн генерации для класса 4 (Shoplifting) запрограммирован следующим образом:
1. Первая фаза (`< 40%` времени): Ключевые точки запястий выдвигаются вперед (имитация того, что человек тянется к витрине).
2. Вторая фаза (`>= 40%` времени): Ключевые точки запястий **очень быстро** возвращаются к ключевым точкам бедра (имитация убирания товара в карман/под куртку).

## Пошаговое руководство: Как обучить ИИ на своих видео

Обучение происходит локально прямо на вашем оборудовании внутри Docker-контейнера `worker`.

### Шаг 1: Подготовка данных (Видео)
Вам нужно собрать короткие видео-отрезки (`.mp4` файлы) с различными действиями и разложить их по соответствующим папкам внутри директории `data/dataset/`.

Убедитесь, что у вас создана следующая структура:
```text
camera-detector/data/dataset/
├── 0_normal/       <-- Сюда кидаем видео с обычной ходьбой и работой
├── 1_fight/        <-- Видео с драками
├── 2_bullying/     <-- Видео с буллингом/конфликтами
├── 3_theft/        <-- Видео с кражами из кассы
└── 4_shoplifting/  <-- Видео с кражами с витрин
```
> [!IMPORTANT]
> Обязательно добавьте много видео в `0_normal/`. Если нейросеть не будет знать, как выглядит "нормальное поведение", она начнёт давать ложные срабатывания на обычные движения!

### Шаг 2: Извлечение скелетов (Экстракция)
На этом этапе система прогонит все ваши видео через YOLO-Pose, извлечет 17 ключевых точек скелетов людей на каждом кадре, нормализует их и сохранит в математические матрицы (`X.npy` и `y.npy`).

Выполните команду в терминале:
```bash
docker compose exec worker python scripts/train_lstm.py --extract
```
*(Процесс может занять время в зависимости от количества видео и мощности CPU/GPU).*

### Шаг 3: Обучение модели (Тренировка)
Теперь, когда геометрия скелетов извлечена, мы обучаем саму LSTM-модель распознавать паттерны во времени.

Выполните команду:
```bash
docker compose exec worker python scripts/train_lstm.py --train
```
В процессе обучения вы увидите в консоли прогресс по эпохам и падение функции потерь (Loss). По завершении, новые обученные веса будут сохранены в файл `action_lstm.pt` в корне проекта.

### Шаг 4: Перезапуск системы
Чтобы ИИ-воркер подхватил новые веса (`action_lstm.pt`), его необходимо перезагрузить:
```bash
docker compose restart worker
```

Готово! Теперь система использует вашу кастомную обученную нейросеть.
